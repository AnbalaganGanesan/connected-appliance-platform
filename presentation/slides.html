Design Decisions

Key Trade-Offs

Vendor calls outside transactions

Network I/O never holds a DB connection. Only the short finalization step (persist attempt + samples + update appliance) runs inside a transaction — preventing connection exhaustion under slow vendors.

Canonical metric model

Vendor-specific names (temp_c, temperature_f) never enter the common domain. All stored samples use canonical names (TEMPERATURE/CELSIUS). Historical data is vendor-independent.

Optimistic concurrency

Appliance rows use a version column (JPA @Version). Concurrent updates produce a 409 rather than silent last-writer-wins overwrites.

Modular monolith — no shared repositories

The metrics module communicates with appliance through declared ports (interfaces), never through shared JPA repositories. Boundaries are enforced by code structure, not by deployment topology.
